Core features
🤖
AI Expense Advisor
Groq API (Llama 3) analyses your subscriptions, flags redundant ones, and suggests budget optimizations with context-aware recommendations.
📁
Subscription Records
File handling (.txt / .json) stores all subscriptions, payment history, and renewal schedules. Persists data across sessions.
🔔
Smart Renewal Alerts
Automated reminders and in-app popup notifications for upcoming renewals and payment deadlines — never miss a billing date.
📊
Analytics Dashboard
Interactive charts showing monthly/yearly spending, category breakdowns, and spending trends using Java2D charts.
💡
Waste Detector
AI-powered detection of unused or overlapping subscriptions. Highlights unnecessary spend and calculates potential savings.
🔐
User Auth
Secure login/signup with password hashing (SHA-256), per-user subscription data isolation, and session management.
Architecture overview
Frontend: Java Swing (GUI) — tabbed dashboard with charts, tables, and alert panels
Backend logic: Java classes (OOP) — services, models, controllers, strategy patterns
Database: SQLite via JDBC — users, subscriptions, payment_history, alerts
File handling: BufferedReader/Writer for subscription exports and AI knowledge base
AI API: Groq API (free, fast) → Llama 3.1 for expense analysis over HTTPS + JSON parsing

Why Groq API (recommended over Gemini for beginners)
Groq — Completely free, very fast, simple REST API, Llama 3.1 model is excellent for financial analysis and recommendations. Just send a JSON POST and get text back. Best for beginners.

Gemini — Also free (1M tokens/month), good alternative. Slightly more complex setup with Google credentials. Use if Groq is unavailable in your region.

Polymorphism

Encapsulation

Abstraction

Multithreading

1. Go to console.groq.com → Sign up free

3. Copy and save it in your project as a config constant (never hardcode in public repos)
4. Free tier: 30 req/min, 6000 req/day — more than enough for a lab project
Step 2 — Java code to call Groq API for subscription analysis
// Add to pom.xml: org.json dependency
public class GroqService {
  private static final String API_KEY = "gsk_xxx";
  private static final String URL = "https://api.groq.com/openai/v1/chat/completions";

  public String analyzeSubscriptions(String subscriptionData) {
    String prompt = "You are a financial advisor. Analyze these subscriptions:\n" + subscriptionData + "\nIdentify waste, overlaps, and suggest optimizations.";
    JSONObject body = new JSONObject()
      .put("model", "llama-3.1-8b-instant")
      .put("messages", new JSONArray().put(
        new JSONObject().put("role","user").put("content", prompt)
      ));

    HttpRequest req = HttpRequest.newBuilder()
      .uri(URI.create(URL))
      .header("Authorization", "Bearer " + API_KEY)
      .header("Content-Type", "application/json")
      .POST(BodyPublishers.ofString(body.toString()))
      .build();

    HttpResponse<String> res = client.send(req, BodyHandlers.ofString());
    return new JSONObject(res.body())
      .getJSONArray("choices").getJSONObject(0)
      .getJSONObject("message").getString("content");
  }
}
